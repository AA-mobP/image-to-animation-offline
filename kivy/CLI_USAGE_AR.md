# استخدام واجهة سطر الأوامر (CLI)

## نظرة عامة
يمكنك الآن تشغيل التطبيق من سطر الأوامر (CMD/Terminal) بدون الحاجة للواجهة الرسومية!

## طريقتان للاستخدام

### 1. استخدام `cli.py` (موصى به - أسرع)
```bash
python cli.py --input photo.jpg
```
✅ **الأفضل**: لا يحمل الواجهة الرسومية، أسرع في التشغيل

### 2. استخدام `main.py` مع معاملات
```bash
python main.py --input photo.jpg
```
⚠️ يحمل مكتبات الواجهة الرسومية حتى في وضع CLI

## الاستخدام الأساسي

### 1. صورة واحدة مع الإعدادات الافتراضية
```bash
python cli.py --input photo.jpg
```

### 2. معالجة مجلد كامل
```bash
python cli.py --input ./images --output ./videos
```

### 3. تخصيص كامل للإعدادات
```bash
python cli.py -i image.png -o ./output -fr 30 -os 2 -bs 3 -d 2 -dc -tp -fs 8 -h264
```

## جميع المعاملات المتاحة

| المعامل | الاختصار | الوصف | القيمة الافتراضية |
|---------|----------|-------|-------------------|
| `--input` | `-i` | مسار الصورة أو المجلد (مطلوب) | - |
| `--output` | `-o` | مجلد الإخراج | `./output` |
| `--split-len` | `-sl` | طول التقسيم (قيمة محددة) | اختيار تلقائي |
| `--split-len-range` | `-slr` | مدى طول التقسيم (مثال: `10-40`) | - |
| `--frame-rate` | `-fr` | معدل الإطارات | `24` |
| `--obj-skip` | `-os` | معدل تخطي الكائنات | `2` |
| `--bck-skip` | `-bs` | معدل تخطي الخلفية | `3` |
| `--duration` | `-d` | مدة عرض الصورة النهائية (بالثواني) | `1` |
| `--draw-color` | `-dc` | رسم بالألوان من البداية | `False` |
| `--two-pass` | `-tp` | استخدام الرسم على مرحلتين | `False` |
| `--fill-speed` | `-fs` | سرعة التعبئة | `5` |
| `--h264` | - | تحويل إلى H264 | `False` |
| `--end-color` | `-ec` | إظهار الصورة الملونة في النهاية | `True` |
| `--gui` | - | فرض تشغيل الواجهة الرسومية | `False` |

### 🎯 ملاحظة مهمة عن `split-len`:
- البرنامج يحسب تلقائياً قائمة التقسيمات الممكنة بناءً على أبعاد الصورة
- القيم الشائعة: `1, 2, 4, 5, 8, 10, 16, 20, 40, 80`
- **استخدم `-slr` (range)** للحصول على **أصغر قيمة متاحة** في المدى المحدد
  - مثال: `-slr 10-40` سيختار `10` إذا كانت متاحة، وإلا سيختار أقرب قيمة لـ `10` في المدى
- **استخدم `-sl` (single)** لتحديد قيمة معينة (سيختار البرنامج الأقرب إذا لم تكن متاحة)
- إذا لم تحدد أي منهما، سيختار البرنامج قيمة مثالية تلقائياً

### 📐 أمثلة على `split-len`:
```bash
# اختيار تلقائي (موصى به)
python cli.py -i photo.jpg

# مدى (سيختار أصغر قيمة متاحة في المدى - الأقرب للحد الأدنى)
python cli.py -i photo.jpg -slr 10-40
# النتيجة: سيختار 10 إذا كانت متاحة، وإلا 16 أو 20

# قيمة محددة (سيختار الأقرب إذا لم تكن متاحة)
python cli.py -i photo.jpg -sl 20
```

## أمثلة عملية

### مثال 1: معالجة سريعة لصورة واحدة
```bash
python cli.py -i photo.jpg
```
النتيجة: ستجد الفيديو في مجلد `output`

### مثال 2: معالجة مجلد بإعدادات مخصصة
```bash
python cli.py -i ./my_images -o ./my_videos -fr 30 -slr 10-40 -tp -dc
```
- معالجة جميع الصور في `my_images`
- حفظ النتائج في `my_videos`
- معدل إطارات 30
- اختيار أفضل split-len بين 10 و 40
- رسم على مرحلتين
- رسم بالألوان

### مثال 3: جودة عالية مع تحويل H264
```bash
python cli.py -i image.png -fr 60 -slr 20-80 -os 1 -bs 1 -fs 10 -h264
```
- معدل إطارات عالي (60 fps)
- split-len عالي (20-80) لتفاصيل أفضل
- تفاصيل أكثر (skip rates منخفضة)
- سرعة تعبئة عالية
- تحويل نهائي إلى H264

### مثال 4: فتح الواجهة الرسومية (الوضع الافتراضي)
```bash
python main.py
# أو
python main.py --gui
```

## ملاحظات مهمة

1. **الوضع الافتراضي**: 
   - `cli.py` يتطلب دائماً `--input`
   - `main.py` بدون معاملات يفتح الواجهة الرسومية
2. **المسارات**: يمكنك استخدام مسارات نسبية أو مطلقة
3. **أنواع الصور المدعومة**: `.png`, `.jpg`, `.jpeg`, `.webp`
4. **المعالجة الدفعية**: عند تحديد مجلد، سيتم معالجة جميع الصور تلقائياً

## استكشاف الأخطاء

### المشكلة: "Error: Input path does not exist"
**الحل**: تأكد من صحة المسار المدخل

### المشكلة: "No image files found in the folder"
**الحل**: تأكد من وجود صور بالصيغ المدعومة في المجلد

### المشكلة: التطبيق يفتح الواجهة الرسومية بدلاً من CLI
**الحل**: استخدم `cli.py` بدلاً من `main.py`

## الحصول على المساعدة
```bash
python cli.py --help
```
